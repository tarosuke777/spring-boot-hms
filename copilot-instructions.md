# Copilot Instructions

## テストクラス作成時の注意点

- Spring テストで `@Sql` を使う場合、テストクラス名に対応した SQL ファイルが存在するか確認する。
  - デフォルトではテストクラスと同じパッケージ配下のクラスパスにある SQL ファイルを探すため、ファイルが存在しないとテストが失敗する。
- テスト専用のデータセットを外部ファイルで管理する場合は、`src/test/resources` 配下に正しいパスで配置する。
- テストコードを追加した後は、対象のテストのみを実行して修正が正しく反映されることを確認する。
- Git/PR の作業では、意図しない未コミット変更がないか確認する。
  - `gh pr create` 実行時に警告が出る場合は、PR に含める差分とワークツリーの残り差分を整理する。

## 追加メモ

- テストクラスと対応する SQL フィクスチャは、同じ機能単位でコミットし、差分の追跡をしやすくする。
- PR 作成前に `git status` でワークツリーの状態を確認し、不要なログや一時ファイルが含まれていないか確認する。
